# 📖 小龙虾 AI 团队 - 完整使用指南

## 🎯 快速开始

### 方式 1：使用自动化脚本（推荐）

双击运行：`清理并重新打包.bat`

这个脚本会自动：
1. 关闭所有运行中的应用实例
2. 清理旧的构建文件
3. 重新打包应用
4. 提示是否立即运行

### 方式 2：手动操作

```bash
# 1. 关闭所有应用实例
任务管理器 → 结束"小龙虾AI团队"进程

# 2. 清理构建目录
rmdir /s /q dist-electron .next out

# 3. 重新打包
npm run electron:build

# 4. 运行
dist-electron\win-unpacked\小龙虾AI团队.exe
```

## 🐛 常见问题解决

### 问题 1：打开超级多窗口

**原因：** 使用了旧版本

**解决：**
1. 关闭所有实例
2. 使用最新打包的版本
3. 确认文件修改时间是最新的

### 问题 2：无法加载本地资源

**错误信息：**
```
Not allowed to load local resource: file:///.../app.asar/out/index.html
```

**原因：** 路径配置问题

**解决：**
1. 确认使用了最新的修复版本
2. 检查 `app.asar.unpacked/out/` 目录是否存在
3. 如果不存在，重新运行 `清理并重新打包.bat`

### 问题 3：打包失败 - 文件被占用

**错误信息：**
```
remove d3dcompiler_47.dll: Access is denied
```

**原因：** 应用仍在运行

**解决：**
```bash
# 方式 1：使用脚本
清理并重新打包.bat

# 方式 2：手动关闭
任务管理器 → 结束所有"小龙虾AI团队"进程
等待 3 秒
重新运行 npm run electron:build
```

### 问题 4：WebSocket 连接失败

**症状：** 应用启动但无法与 Agent 通信

**解决：**
1. 检查端口 3001 是否被占用
2. 查看控制台日志
3. 确认 `server/ws-server.js` 是否正常启动

### 问题 5：API Key 配置丢失

**原因：** 设置存储在本地

**解决：**
1. 重新进入设置页面
2. 配置 API Key
3. 测试连接
4. 保存设置

## 📁 文件结构说明

### 开发目录
```
apps/web/
├── electron/              ← Electron 主进程
│   ├── main.cjs          ← 主进程入口
│   └── preload.cjs       ← 预加载脚本

│   └── lib/              ← 工具函数

```
小龙虾AI团队.exe
└── resources/
    ├── app.asar                    ← 主应用代码（压缩）

    │   ├── out/                    ← Next.js 静态文件 ✅
    │   │   ├── index.html
    │   │   └── _next/
    │   └── node_modules/           ← 原生模块
    └── server/                     ← WebSocket 服务器

## 🔧 修复历史

### 修复 1：多窗口问题
- **问题：** 双击打开多个窗口
- **原因：** 缺少单实例锁
- **修复：** 添加 `app.requestSingleInstanceLock()`
- **状态：** ✅ 已修复

### 修复 2：路径加载错误
- **问题：** 无法加载本地资源
- **原因：** 路径配置错误，文件在 asar 中
- **修复：** 使用 `asarUnpack` + `process.resourcesPath`
- **状态：** ✅ 已修复

### 修复 3：API 路由冲突
- **问题：** 静态导出与 API 路由不兼容
- **原因：** `dynamic = "force-dynamic"` 声明
- **修复：** 移除 API 路由，使用 WebSocket
- **状态：** ✅ 已修复
