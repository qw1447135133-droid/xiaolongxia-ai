# 🔧 多窗口问题修复报告

## 问题描述

**症状：** 双击 `小龙虾AI团队.exe` 后会打开超级多的窗口

## 根本原因分析

### 1. Next.js 配置问题
- **问题：** Next.js 没有配置静态导出（`output: 'export'`）
- **影响：** 生产环境下 Electron 无法找到 `out/index.html` 文件
- **后果：** 加载失败导致应用重复尝试启动

### 2. Electron 路径错误
- **问题：** 主进程中的路径配置错误
  ```javascript
  // 错误的路径
  `file://${path.join(__dirname, 'out', 'index.html')}`

  // 正确的路径
  `file://${path.join(__dirname, '..', 'out', 'index.html')}`
  ```
- **影响：** 无法正确加载 HTML 文件

### 3. 缺少多实例保护
- **问题：** 没有使用 `app.requestSingleInstanceLock()`
- **影响：** 允许多个应用实例同时运行
- **后果：** 每次点击都会创建新窗口

### 4. 窗口重复创建
- **问题：** `createMainWindow()` 函数没有检查窗口是否已存在
- **影响：** 可能创建多个主窗口

## 修复方案

### ✅ 修复 1：配置 Next.js 静态导出

**文件：** `next.config.js`

```javascript
/** @type {import('next').NextConfig} */
const nextConfig = {
  reactStrictMode: true,
  output: 'export',           // 启用静态导出
  distDir: 'out',             // 输出到 out 目录
  images: {
    unoptimized: true,        // 禁用图片优化（静态导出要求）
  },
};
module.exports = nextConfig;
```

**效果：**
- Next.js 会生成静态 HTML 文件到 `out` 目录
- Electron 可以正确加载这些文件

### ✅ 修复 2：添加单实例锁

**文件：** `electron/main.cjs`

```javascript
// 防止多实例启动
const gotTheLock = app.requestSingleInstanceLock();
if (!gotTheLock) {
  console.log('[main] Another instance is already running, quitting...');
  app.quit();
  process.exit(0);
}
```

**效果：**
- 只允许一个应用实例运行
- 第二次点击会聚焦到已有窗口

### ✅ 修复 3：修正文件路径

**文件：** `electron/main.cjs`

```javascript
const url = isDev()
  ? 'http://localhost:3000'
  : `file://${path.join(__dirname, '..', 'out', 'index.html')}`;
```

**效果：**
- 正确定位到打包后的 HTML 文件
- 避免 404 错误

### ✅ 修复 4：防止窗口重复创建

**文件：** `electron/main.cjs`

```javascript
function createMainWindow() {
  // 防止重复创建
  if (mainWindow && !mainWindow.isDestroyed()) {
    console.log('[main] Window already exists, focusing...');
    mainWindow.focus();
    return;
  }
  // ... 创建窗口逻辑
}
```

**效果：**
- 检查窗口是否已存在
- 避免创建多个主窗口

### ✅ 修复 5：处理第二实例

**文件：** `electron/main.cjs`

```javascript
app.on('second-instance', () => {
  console.log('[main] Second instance detected, focusing main window');
  if (mainWindow) {
    if (mainWindow.isMinimized()) mainWindow.restore();
    mainWindow.focus();
  }
});
```

**效果：**
- 当用户再次点击应用时，聚焦到已有窗口
- 提供更好的用户体验

### ✅ 修复 6：改进窗口关闭行为

**文件：** `electron/main.cjs`

```javascript
let isQuitting = false;

mainWindow.on('close', (e) => {
  if (!isQuitting) {
    e.preventDefault();
    mainWindow.hide();
  }
});

app.on('before-quit', () => {
  isQuitting = true;
  // 清理逻辑
});
```

**效果：**
- 关闭窗口时隐藏而不是退出（类似 macOS 行为）
- 真正退出时才清理资源

## 技术细节

### 为什么需要静态导出？

Electron 打包后的应用是一个独立的可执行文件，无法运行 Next.js 的服务器端渲染。因此需要：

1. 将 Next.js 应用导出为静态 HTML/CSS/JS
2. Electron 通过 `file://` 协议加载这些静态文件
3. API 路由通过独立的 WebSocket 服务器处理

### 为什么路径是 `__dirname/../out`？

打包后的目录结构：
```
小龙虾AI团队.exe
└── resources/
    └── app.asar
        ├── electron/
        │   └── main.cjs  ← __dirname 在这里
        └── out/          ← HTML 文件在这里
            └── index.html
```

所以需要 `path.join(__dirname, '..', 'out', 'index.html')`

### 单实例锁的工作原理

1. 第一个实例调用 `app.requestSingleInstanceLock()`，获得锁
2. 第二个实例调用时，返回 `false`，立即退出
3. 第一个实例收到 `second-instance` 事件，可以聚焦窗口

## 常见问题

### Q: 为什么不直接禁止多次点击？
**A:** 用户可能会多次点击，这是正常行为。应该优雅处理，而不是报错。

### Q: 为什么关闭窗口不退出应用？
**A:** 这是桌面应用的常见模式（尤其是 macOS）。用户可以从任务栏完全退出。

### Q: WebSocket 服务器会重复启动吗？
**A:** 不会。单实例锁确保只有一个应用实例，因此只有一个 WS 服务器。
